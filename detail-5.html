运营团队 · 效能提升项目

业务团队信息检索 Agent
切实助力工作提效

为运营团队打造一个 "问对人、找对文档" 的智能入口， 把跨部门 / 跨代理商 / 跨文档的信息检索，从 "靠人问、凭记忆" 变成 "问 Agent、有出处"。

一
项目背景
信息分散、协作摩擦大，隐性时间成本高

运营团队在日常工作中需高频对接 财务、产品、法务、设计、数据分析 等多个职能部门， 同时还要协调外部代理商，处理数量庞大的合同与业务文档。信息分散在不同系统、不同人手里，带来三类典型痛点：

痛点 01

"该找谁"不清晰

一个具体问题（如某类退款审批、某活动的数据口径、某合同条款解释）到底归属哪个部门、对接人是谁，新人和老人都常常需要反复打听。

痛点 02

"看哪个文档"不确定

制度、SOP、模板、合同散落在多个位置，版本多、更新不同步，容易看到过期或错误的版本。

痛点 03

咨询与查询成本高

大量时间消耗在"找人 — 等回复 — 再推进"的链路上，跨部门等待成本尤其高，重复问题反复被咨询也拉低了被咨询方的效率。

本质问题

团队缺少一个 统一、可信、随时可用 的信息检索入口，导致隐性时间成本高、协作摩擦大、知识沉淀依赖个人。

二
项目目标
优先问 Agent，拿到有出处的答案
总目标：搭建业务信息检索 Agent，让运营同学遇到问题时优先问 Agent， 实现 "问对人、找对文档、拿到有出处的答案"，降低跨部门咨询成本，沉淀团队知识资产。

总目标拆解为四个可衡量的子目标：

子目标 衡量方式 目标水平 说明
01问对人 高频问题的归属 / 对接人命中率  100% 常见问题能直接给出归属部门与对接人，无需再人工打听
02找对文档 返回最新版本文档的准确率  100% 制度 / SOP / 模板 / 合同定位到正确位置与现行版本
03有出处 答案附带来源的比例  100% 每条业务答案均注明文档 / 位置或对接部门 / 对接人
04降成本 高频问题的咨询等待时间 <=1min 常见问题即问即答，减少"找人—等回复"的链路消耗

覆盖度方面，需优先命中运营高频问题，例如：

Q某家代理商签的主合同编号是多少？对应的补充协议编号是？
Q结算单的财务问题联系谁？对接的法务老师是哪位？
Q某项制度 / SOP / 模板 / 口径说明，最新版本在哪里？
三
项目行动
建知识库 · 强化规则 · 试用迭代

3.1　建立运营团队业务知识库

按 "运营同学实际会问什么" 的视角组织内容，而非按部门堆文档。建议知识库分为四层：

1

问题 — 归属映射

常见问题 → 归属部门 → 对接人 → 备用对接人 → 响应时效 / 对接方式。

2

文档索引

制度 / SOP / 模板 / 口径说明，标注文档名、位置 / 链接、适用场景、版本与更新日期、维护人。

3

合同与代理商专区

各代理商对接人、合作范围、关键合同条款要点与常见问答、合同存放位置。

4

口径与术语库

数据指标口径、业务术语、审批流程等易产生歧义的定义。

关键设计

知识库以文件形式沉淀，放置在 Agent 的工作目录下，确保 Agent 每次都能读取到最新版本。

3.2　在 System Prompt 中强化"先检索后回答"规则

Agent 的核心约束写入 system prompt，关键规则包括：

01强制检索

回答任何业务问题前，必须先检索工作目录下的知识库，不得凭记忆或臆测作答。

02必带出处

答案需注明来源（文档名 + 位置 / 链接，或对接部门 + 对接人）。

03诚实兜底

查不到时明确告知"未收录"，并给出最可能的归属方向或建议的下一步，而非编造。

04区分事实与建议

涉及法务 / 财务的正式结论，引导用户以官方文档或对接人确认为准。

3.3　试用与迭代

Agent 已完成搭建并投入试用。试用阶段的重点是验证检索命中率、答案准确性与实际提效效果，并据此补全知识库盲区。
四
项目复盘
项目成果

✓"先检索后回答"规则有效

强制检索 + 必带出处，显著降低了 Agent 编造答案的情况，答案可追溯，用户更敢用、更信任。

✓以"业务实际问题"建库

贴合运营同学真实提问习惯，检索命中更自然。

✓放工作目录、随取随读

更新后 Agent 立即生效，避免了"模型记忆过期"的问题。

五
后续规划
部署到企业微信机器人，从"个人使用"升级为"团队随手可用"

▶部署到企业微信机器人

将 Agent 接入企微机器人，让能力从 "个人使用" 升级为 "团队随手可用"。具体落地：

STEP 01 · 建群入群

新建运营团队专属企微群，把机器人拉进群，成员 @机器人 即可提问，答案群内可见，形成公开的问答沉淀。

STEP 02 · 复用规则

机器人复用同一套知识库与 system prompt 规则（先检索、带出处、诚实兜底），保证群内答案与个人使用一致。

STEP 03 · 明确边界

明确使用方式与边界：涉及法务 / 财务的正式结论，仍引导以官方文档或对接人确认为准。
